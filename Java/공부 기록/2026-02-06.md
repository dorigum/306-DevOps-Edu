# Java enum(열거형)
#### 1. enum 이란?
: `enum`은 열거형(enumeration)의 줄임말로, **서로 관련된 상수들의 집합**을 정의할 수 있는 Java의 특별한 데이터 타입이다.
```java
pulbic enum Week {
	MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}
```
- 위 예시처럼, `Week`은 요일을 의미하는 7개의 상수로 구성된 열거형
- 각 값들은 `Week.MONDAY`, `Week.TUESDAY`처럼 접근
- Enum은 싱글톤처럼 동작하며, JVM에 한번만 로드
- DB값과 매핑할 때는 `@Enumerated(EnumType.STRING)`같은 JPA 어노테이션 사용 가능

#### 2. 왜 enum을 사용할까?

| 기존 방식 (`final static`)                | enum 방식                |
| ------------------------------------- | ---------------------- |
| `public static final int MONDAY = 0;` | `MONDAY`와 같은 이름을 직접 정의 |
| 타입 안정성 부족                             | 컴파일 시 타입 체크 가능         |
| 잘못된 값도 사용될 수 있음                       | 정의된 값만 사용 가능           |
| 기능 확장 어려움                             | 메소드 정의 가능, 인터페이스 구현 가능 |

#### 3. enum 사용 예제

##### 1. 기본 사용
```java
public enum Direction {
	EAST, WEST, SOUTH, NORTH
}
Direction d = Direction.EAST;
```

##### 2. switch문과 함께 사용
```java
switch(d) {
	case EAST -> System.out.print("동쪽");
	case WEST -> System.out.print("서쪽");
}
```

##### 3. 필드와 메소드를 가진 enum
```java
public enum Grade {
    BASIC(0), SILVER(1), GOLD(2);

    private final int level;

    Grade(int level) {
        this.level = level;
    }

    public int getLevel() {
        return level;
    }
}
// 사용 예
Grade g = Grade.GOLD;
System.out.println(g.getLevel());  // 출력: 2

```
- 열거형의 생성자는 묵시적으로 `private`이다. 외부에서 객체 생성 불가

##### 4. enum이 유용한 대표적인 사용 사례
| 분야        | 사용 예시                                          |
| --------- | ---------------------------------------------- |
| 날짜/시간     | 요일 (`Week`), 월 (`Month`)                       |
| 상태 코드     | 주문 상태 (`ORDERED`, `SHIPPED`, `DELIVERED`)      |
| 게임        | 방향 (`UP`, `DOWN`, `LEFT`, `RIGHT`), 등급         |
| 애플리케이션 로직 | 사용자 역할 (`ADMIN`, `USER`, `GUEST`)              |
| API 처리    | HTTP 상태 코드 (`OK`, `NOT_FOUND`, `SERVER_ERROR`) |

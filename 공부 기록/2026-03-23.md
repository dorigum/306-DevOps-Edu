# 웹 프로그래밍 언어
- 웹 브라우저에서 실행되는 언어를 의미한다.
- 대표적으로 **HTML(구조), CSS(표현), JavaScript(동작)** 3가지가 있다.
- 이 세 언어를 조합하여 **웹 페이지와 웹 애플리케이션**을 구현한다.
- 브라우저가 직접 해석할 수 있는 언어는 이 3가지 뿐이며, 서버 측 언어(Java, PHP, Python 등)는 브라우저가 아니라, **WAS(Web Application Server)** 가 처리한다.

## 1. Web FrontEnd
### 1.  HTML(HyperText Markup Language)
- 웹 브라우저에 표시되도록 설계된 문서의 표준 마크업 언어
- 웹 페이지의 GUI(화면 구성) 를 담당
	- 예: 로그인 화면, 회원가입 폼, 게시판 목록, 방명록, 장바구니 화면 등
- 사용자의 입력 값(Form)을 받거나, 서버에서 전달받은 결과를 화면에 출력하는 역할

### HTML vs XML
- **XML**
	- 환경 설정 문서
	- 데이터 전송 포맷
		- 데이터 양이 많아지면서 버그 발생 가능성이 커짐
- **HTML**

#### HTML 버전의 발전
- 과거:  HTML4
- 현재:  HTML5(표준)
	- HTML5는 단순 화면 구성 뿐만 아니라 다양한 웹 API 제공
		- Web Storage(로컬 저장소)
		- Drag & Drop
		- Audio / Video 재생
		- Web Socket(실시간 통신)
		- Canvas / SVG(그래픽 처리)
- HTML5 전에는 HTML이 단순 정보를 보여주는 것이었다면, HTML5 이후 HTML은 웹 애플리케이션 개발을 위한 핵심 기술로 자리 잡음

#### HTML 문법 특징
- ※ 마크업 언어는 오타가 나거나 형식이 잘못 되어도 실행이 되기 때문에, 어떤 부분에서 오류가 발생한 건지 파악하기 어렵고, 불필요한 코드가 많아진다.
- 대, 소문자 구분하지 않음
- 마크업 언어이므로 태그(< >) 를 이용해 구조를 표현

#### 정리
- HTML은 웹 페이지의 구조와 화면을 담당하는 기본 언어
- 현재 표준은 HTML5, 다양한 API 제공
- 문법이 관대하여 오류가 있어도 실행은 되지만, 표준을 지키는 것이 중요

# `<section>` vs `<article>` 차이

- `section` 은 **주제 별 구역**
- `article`은 **독립적으로 떼어내도 의미가 있는 완결된 내용**

### 1. 먼저 감각적으로 이해하기

예를 들어 뉴스 사이트를 생각해보자. 페이지에 이런 내용이 있다고 하면

- 정치 뉴스 구역
- 경제 뉴스 구역
- 스포츠 뉴스 구역

이런 “주제 별 구역”은 `section`에 가깝다.

그 안에 들어가는 개별 뉴스 하나하나는:

- 제목이 있고
- 본문이 있고
- 단독으로 공유하거나 복사해도 의미가 있음

이런 것은 `article`에 가깝다.

즉,

- **section = 내용을 주제 별로 묶는 구역**
- **article = 하나의 독립된 글/게시물/콘텐츠**

---
## 2. section 태그란?

`section`은 문서 안에서 **관련 있는 내용을 주제 별로 나누는 영역**.

보통 이런 경우에 쓴다.

- 소개 영역
- 공지사항 영역
- 강의 목록 영역
- 문의하기 영역

즉, 페이지를 큰 주제 단위로 나누고 싶을 때 쓰는 태그

예시:

```jsx
<section>
  <h2>공지사항</h2>
  <p>이번 주는 서버 점검이 있습니다.</p>
</section>

<section>
  <h2>강의 목록</h2>
  <ul>
    <li>HTML 기초</li>
    <li>CSS 기초</li>
  </ul>
</section>
```

여기서 각 `section`은 “주제 별 구역”

중요한 포인트는

`section` 자체는 보통 **독립된 콘텐츠**라기보다,

문서 안의 **한 부분**이라는 점

---
## 3. article 태그란?

`article`은 **그 자체로 독립적으로 완결된 콘텐츠**를 뜻함.

예를 들면:

- 뉴스 기사 1개
- 블로그 글 1개
- 게시판 글 1개
- 댓글 1개
- 제품 리뷰 1개
- 포럼 글 1개

이런 것들은 페이지에서 떼어내도 의미가 있다.

예시:

```jsx
<article>
  <h2>HTML5 시맨틱 태그 정리</h2>
  <p>HTML5에서는 문서 구조를 더 명확하게 표현할 수 있습니다.</p>
</article>
```

이 글 하나를 메일로 보내든, RSS로 뽑든, 다른 페이지에 재사용하든

그 자체로 하나의 콘텐츠가 된다.

그래서 `article`

---
## 4. 가장 중요한 차이

## section

“문서의 일부를 주제별로 나눈 구역”

## article

“독립적으로 배포하거나 재사용할 수 있는 완결된 콘텐츠”

즉,

`section`은 **구성 단위**이고,

`article`은 **독립 콘텐츠 단위**라고 보면 된다.

---
## 5. 둘이 왜 비슷해 보이냐?

겉모습만 보면 둘 다 그냥 박스처럼 보이기 때문이다.

브라우저 화면에서 `section`과 `article`은 기본적으로 거의 비슷하게 보인다.

차이는 **디자인이 아니라 의미**에 있다.

즉, 이 둘은 “어떻게 보이느냐”보다

“이 내용이 어떤 성격이냐”를 나타내는 태그예요.

그래서 CSS 없이 보면 정말 비슷해 보이는 게 맞다.

---
## 6. section 안에 article 넣을 수 있나?

네, **넣을 수 있다**

오히려 아주 자연스러운 구조.

예를 들어 블로그 메인 페이지:

- 최신 글 section
- 인기 글 section

이건 각각 `section`

그 안에 있는 글 하나하나는 `article`

예시:

```jsx
<section>
  <h2>최신 글</h2>

  <article>
    <h3>자바스크립트 배열 정리</h3>
    <p>배열 메서드를 쉽게 정리해봅시다.</p>
  </article>

  <article>
    <h3>리눅스 기본 명령어</h3>
    <p>자주 쓰는 명령어를 예제로 배워봅시다.</p>
  </article>
</section>
```

이 구조는 아주 적절.

왜냐하면:

- `최신 글`은 하나의 **주제 구역** → `section`
- 각 글은 **독립 콘텐츠** → `article`

---
## 7. article 안에 section 넣을 수 있나?

이것도 **가능하다.**

예를 들어 하나의 긴 블로그 글(`article`) 안에:

- 소개
- 예제
- 결론

처럼 내부 주제별 구간을 나눌 수 있다.

예시:

```jsx
<article>
  <h1>HTML 시맨틱 태그 정리</h1>

  <section>
    <h2>시맨틱 태그란?</h2>
    <p>의미를 가진 태그를 말합니다.</p>
  </section>

  <section>
    <h2>section과 article의 차이</h2>
    <p>section은 구역, article은 독립 콘텐츠입니다.</p>
  </section>
</article>
```

이것도 아주 자연스럽다.

왜냐하면:

- 전체 글 하나는 독립 콘텐츠 → `article`
- 글 내부의 소주제 구간 → `section`

# HTML5에서 **“boolean attribute (불리언 속성)”** 개념

> ✅ `<input disabled>` 와 `<input disabled="disabled">` 는 **완전히 동일하게 동작**
> 
> ⭐ HTML5에서는 `<input disabled>`가 더 표준적이고 권장됨

# 1. 왜 두 가지 방식이 존재하냐?

이건 역시 **HTML vs XHTML 역사 때문**입니다.

과거 XHTML에서는 아래와 같이 **속성 = 값 형태를 반드시 맞춰야 했다.**

```jsx
<input disabled="disabled">
```

# 2. HTML5에서는 어떻게 바뀌었나?

HTML5에서는 `disabled`, `checked`, `selected` 같은 속성은

**“값이 필요 없는 속성”** 으로 정의된다.

👉 이런 걸 **boolean attribute (불리언 속성)**이라고 한다.

# 3. boolean attribute의 핵심 규칙

이건 매우 중요

> ❗ 값이 있느냐가 아니라 “속성이 존재하느냐”가 중요

즉:

```jsx
<input disabled>
```

이건

```jsx
<input disabled="disabled">
<input disabled="true">
<input disabled="false">
```

👉 전부 **disabled = 적용됨 (true)**

# 4. 그래서 `<input disabled>`가 더 좋은 이유

HTML5에서는 이렇게 쓰는 게 더 맞다.

```jsx
<input disabled>
```

이유:

- 더 간결함
- HTML5 표준 스타일
- 의미 전달이 명확함

# 5. 그럼 언제 `"disabled"`를 쓰냐?

요즘 HTML에서는 거의 안 쓴다.

하지만 아래 상황에서는 볼 수 있다.

### ✔ JSX / React

```jsx
<input disabled={true} />
```

# 6. boolean attribute 대표 예시

아래는 전부 같은 규칙을 따릅니다:

```jsx
<input disabled>
<input checked>
<option selected>
<input readonly>
```

# HTML5에서 content가 없을 때 닫는 태그를 안해도 되는가?

> ✅ HTML5에서는 `<input>` 이렇게만 써도 된다.
> 
> ❌ `<input />`는 “틀린 건 아니지만 필수는 아님”

---

# 1. 왜 이런 헷갈림이 생기냐?

이건 **HTML vs XHTML 문법 차이** 때문이다.

과거에는:

- HTML → 느슨한 문법
- XHTML → 엄격한 XML 문법

이렇게 나뉘어 있었다.

---

# 2. HTML5 기준 (지금 기준 👍)

HTML5에서는 `input` 같은 태그는 **“void element (빈 요소)”** 라고 한다.

즉,

- 내용(content)이 없음
- 닫는 태그가 애초에 존재하지 않음

대표적인 태그들:

```jsx
<input>
<img>
<br>
<hr>
<meta>
<link>
```

이 태그들은 이렇게 쓰는 게 **정석**입니다:

```jsx
<input type="text">
<img src="image.jpg">
<br>
```

👉 닫는 태그 없음

👉 / 필요없음

## 3. 그럼 `<input />`는 뭐냐?

이건 **XHTML 스타일 문법이다**.

```jsx
<inputtype="text"/>
```

HTML5에서도 이 문법을 허용하긴 한다..

그래서 에러는 아니다.

하지만 중요한 포인트:

> HTML5에서는 `/`가 아무 의미 없음 (그냥 무시됨)

즉,

- `<input>` = `<input />` (완전히 동일하게 동작)

## 4. 언제 `/`를 쓰냐?

### ✔ 보통 이런 경우에 씀

## 6. 절대 하면 안 되는 것 ❌

이건 진짜 중요하다.

```jsx
<input></input>   ❌
```

왜냐하면:

- `input`은 **닫는 태그가 존재하지 않는 태그**
- 이렇게 쓰면 브라우저가 이상하게 해석할 수 있음

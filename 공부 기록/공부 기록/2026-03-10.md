# Professional Java Console-based Kiosk System Architecture

본 설계서는 GUI가 없는 콘솔 환경에서 MVC 패턴과 계층별 아키텍처(Layered Architecture)를 통해 시스템의 확장성과 유지보수성을 극대화하는 데 중점을 둡니다.

## 1. 페르소나 (Persona) & 사용자 경험 (UX)

콘솔 환경에서도 타겟층인 2030의 편의성을 위해 **'입력 최소화'** 를 목표로 합니다.

- **사용자 (김지훈, 24세):** "콘솔창이라도 복잡한 입력은 싫어요. 번호 하나로 제 단골 메뉴를 바로 주문하고 싶습니다."

- **관리자 (이영희, 45세):** "텍스트로 보더라도 우리 매장의 오늘 매출 현황이 한눈에 표(Table) 형태로 정리되어 보였으면 좋겠어요."


## 2. 시스템 아키텍처 (MVC Pattern)

콘솔 프로젝트의 패키지 구조를 실무 수준으로 분리합니다.

- **Controller (`controller`):** 사용자의 메뉴 선택 입력을 받아 해당하는 Service를 호출하고 결과를 View에 전달.

- **Service (`service`):** 비즈니스 로직(포인트 계산, 매출 통계 연산, 재고 체크 등) 처리.

- **Model (`model`):**
    - **DTO/VO:** 데이터 이동을 위한 객체 (`UserDTO`, `OrderDTO`).
    - **DAO:** 데이터베이스(MySQL)와 직접 통신하며 CRUD 수행.

- **View (`view`):** `System.out.println` 및 `Scanner`를 담당하는 클래스. 메뉴 출력 및 사용자 입력 수신.

## 3. 데이터베이스 설계 (ERD)
### Table: ADMIN - 매장 관리용

- `admin_id` (PK, VARCHAR): 관리자 로그인 ID
- `admin_pw` (VARCHAR): 비밀번호 (암호화 권장)
- `store_name` (VARCHAR): 매장명
- `store_address` (VARCHAR): 매장 주소

### Table: USER - 고객 관리용

- `user_tel` (PK, VARCHAR): 전화번호 (회원 식별키)
- `user_pw` (VARCHAR): 간편 비밀번호 (4자리 등)
- `point_balance` (INT): 현재 보유 포인트
- `fav_menu_id` (FK, INT): 퀵오더용 선호 메뉴 ID

### Table: PRODUCT & ORDERS

- `PRODUCT`: 메뉴 아이디, 카테고리, 이름, 가격, 재고수량, 판매상태
- `ORDERS`: 주문번호, 주문자(Tel), 매장ID, 총금액, 사용포인트, 주문일시
- `ORDER_ITEMS`: 주문번호별 메뉴 리스트 및 선택 옵션 상세 기록


## 4. 핵심 로직 플로우 차트 (Flowchart)
### 포인트 적립 및 사용 상세 로직

1. **결제 요청:** 장바구니 총액($Sum$) 계산.
    
2. **회원 조회:** 전화번호 입력 -> DB에서 `USER` 정보 호출.
    
3. **포인트 체크:** - `if (user.point_balance >= 1000)` 일 때만 "포인트를 사용하시겠습니까? (Y/N)" 출력.
    
    - **Y 선택 시:** `Total_Pay = Sum - user.point_balance`. DB에 포인트 0으로 업데이트.
    - **N 선택 시:** `Total_Pay = Sum`.
    
4. **최종 결제 및 적립:**
    - 결제 완료 메시지 출력.
    - **신규 적립:** `New_Point = Total_Pay * 0.1`.
    - **DB 반영:** `UPDATE USER SET point_balance = point_balance + New_Point`.

### 관리자 매출 분석 로직

1. **조회 기간 설정:** 1.일별 2.월별 3.연별 중 선택.
2. **데이터 호출:** `ORDERS` 테이블에서 기간별 `GROUP BY` 쿼리 실행.
3. **통계 계산:** - 총 매출액 수집.
    - `ORDER_ITEMS`와 조인(Join)하여 가장 많이 팔린 메뉴 상위 3개 추출.
    
4. **콘솔 출력:** 깔끔한 표(Table) 형식으로 통계 데이터 대시보드 출력.

## 6. 실무급 기술 요구사항 (Non-Functional)

- **Transaction 관리:** 주문 처리 중 오류 발생 시 모든 DB 작업을 Rollback 처리.
- **Log 기록:** 모든 주문과 포인트 변동 내역을 별도 로그 파일(`.txt`)로 저장하도록 설계.
- **예외 처리:** 잘못된 번호 입력, 재고 부족 시 `CustomException`을 정의하여 처리.

# 자바스크립트 `eval()` 함수와 대체 방법

## 1. `eval()` 함수란?
- **정의**: 문자열(String)로 전달된 코드를 **자바스크립트 코드처럼 실행하는 함수**.

- **초창기 사용 예시**
```jsx
let code = "2 + 3";
console.log(eval(code)); // 5
``` 

- **특징**
    - 문자열을 실행 시점(runtime)에 **자바스크립트 코드로 해석**한다.
    - 동적으로 계산식이나 함수를 실행할 수 있다.
    - 과거에는 간단한 계산기 구현, JSON 파싱(ES5 이전) 등에 자주 사용됨.

---
## 2. `eval()` 함수의 문제점 (단점)

1. **보안 취약점**
    - 외부 입력을 그대로 `eval()`에 넘기면, 악성 코드 실행 가능
```jsx
let userInput = "alert('해킹!')";
eval(userInput); // 공격 코드 실행됨 ❌
```

2. **성능 저하**
    - 자바스크립트 엔진의 최적화를 깨뜨려 속도가 느려짐.

3. **디버깅 어려움**
    - 런타임에 문자열이 코드로 변환되므로 코드 추적/디버깅이 힘듦.

👉 그래서 **현대 개발에서는 사용을 권장하지 않음.**

---

## 3. 대체 방법: `new Function()`
- **정의**: 생성자 함수로 동적 함수를 만들 수 있다.

- **사용 예시**
```jsx
ex) const f = new Function('a', 'b', 'return a + b');
                 // 첫 번째 ~ (n-1)번째 매개변수 : 함수의 인자 이름들
    			 // 마지막 인자 : 함수 본문 (문자열)
    				  
    
    let no1 = 5, no2 = 10, oper = "+";
    let result = new Function("return " + (no1 + oper + no2))(); // 즉시 실행
    console.log(result); // 15
``` 

- **장점**
    - `eval()`처럼 문자열을 코드로 실행 가능.
    - **자신의 독립된 함수 스코프**를 가지므로 `eval()`보다 안전.
    - 성능 상 약간 더 낫다.

---
## 4. `new Function()`의 단점

1. **보안 문제 여전**
    - 외부 입력값을 그대로 넣으면 동일하게 악성 코드 실행 위험.
        
    - 예:
```jsx
let userInput = "alert('해킹!')";
new Function(userInput)(); // 공격 코드 실행 ❌
```

2. **가독성과 유지보수성 저하**
    - 문자열 기반 코드 생성 → 버그 추적 어렵다.

3. **성능 문제**
    - 여전히 런타임에 해석 → 일반 함수 호출보다 느림.

---
## 5. 그러면 언제 사용해야 할까?
- ✅ **가능한 사용하지 않는다.**
- ✅ 하지만 **제한적으로 필요할 때 사용 가능**:
    - 계산식 파싱기, 동적 코드 실행이 꼭 필요한 상황.
    - 예: **수식 계산기**(단, 신뢰할 수 있는 내부 입력일 때만).
- ❌ **사용자 입력을 그대로 넣어서는 절대 안 됨.**

---
## 6. eval() vs new Function() 핵심 차이

## 1) eval() 은 “현재 스코프”에서 실행됨

- `eval()`은 **호출된 위치의 스코프(변수 영역)** 안으로 파고들어와서, 거기에 새로운 변수를 만들거나 기존 변수를 바꿀 수 있다.
- 즉, **외부 코드가 현재 실행 맥락을 직접 침범**하는 것.

```jsx
function test() {
  let secret = 123;
  eval("secret = 999;");
  console.log(secret); // 999  ← eval이 바꿔버림
}
test();
```

👉 **문제점**
- 외부 입력을 그대로 eval에 넣으면, 현재 함수/블록의 변수까지 손댈 수 있다.
- 이건 보안 위험 + 예측하기 힘든 버그의 원인.

---
## 2) new Function() 은 “독립된 함수 스코프”에서 실행됨

- `new Function(code)`로 만든 코드는 **새로운 함수 객체 안에서**만 실행된다.
- 따라서 **현재 스코프의 로컬 변수에는 접근하지 못한다**
- 오직 전역 객체(window, global)와 자신이 가진 매개변수/로컬 변수만 접근할 수 있다.

```jsx
function test() {
  let secret = 123;
  const fn = new Function("return typeof secret;");
  console.log(fn()); // "undefined"  ← 외부 함수의 변수 접근 불가
}

test();
```

👉 **장점**
- 내부 동작이 "샌드박스처럼" 격리되어 있음
- 현재 실행 컨텍스트를 오염시키지 않음.

---
## 3) 비교 정리

|구분|eval()|new Function()|
|---|---|---|
|실행 스코프|**현재 스코프**에서 실행 (로컬 변수 접근/변경 가능)|**새로운 함수 스코프** 안에서 실행 (외부 로컬 변수 접근 불가)|
|보안성|취약 (현재 코드 침범 가능)|상대적으로 안전 (전역 + 매개변수만 사용)|
|성능|JIT(Just-in-Time) 최적화 깨짐, 느림|독립 함수라 엔진 최적화에 더 유리|
|실무 사용|**비권장**|그래도 제한적 사용은 가능|

## 4) 그래서 “eval보다 안전하다”는 뜻은?

- `new Function()`도 문자열 코드를 실행하기 때문에 위험하지만,
- **eval()처럼 호출한 함수의 로컬 변수까지 손대지 않는다**는 점에서 조금 더 안전하다는 뜻이다.

👉 정리하면:
- **eval()** = 현재 방(스코프)에 침입해서 마음대로 바꾸는 도둑.
- **new Function()** = 자기 방(새 함수 스코프)에서만 노는 손님. 남의 집은 못 건드림.

## 6. 안전한 대안

- **JSON 파싱** → `eval()` 대신 `JSON.parse()`
- **동적 실행 로직**
    - if/switch/case, 객체 매핑을 통한 분기 처리
    - 예:
```jsx
let operators = {
  "+": (a, b) => a + b,
  "-": (a, b) => a - b,
  "*": (a, b) => a * b,
  "/": (a, b) => a / b,
};

let result = operators[oper](no1, no2);
console.log(result);
```

# 🌟for in vs for of
- `for...in` 👉 **"키(index, property)"를 순회**
- `for...of` 👉 **"값(value)"을 순회**

---
# 1. for...of (값을 꺼낸다)

```jsx
const arr = ['a', 'b', 'c'];

for (let value of arr) {
  console.log(value);
}
```

👉 결과

```
a
b
c
```

✔ 특징
- 배열에서 **값 자체**를 가져옴
- 우리가 보통 원하는 방식
- `Array`, `String`, `Map`, `Set` 등 iterable에 사용

---
# 🔹 2. for...in (키를 꺼낸다)

```jsx
const arr = ['a', 'b', 'c'];

for (let key in arr) {
  console.log(key);
}
```

👉 결과

```
0
1
2
```

✔ 특징
- 배열의 **index(키)** 를 가져옴
- 사실 배열보다는 **객체(Object)** 에 더 적합

---
# 🔥 차이 확실히 보기

```jsx
const arr = ['a', 'b', 'c'];

for (let key in arr) {
  console.log(key, arr[key]);
}
```

👉 결과

```
0 a
1 b
2 c
```

👉 즉,
- `key` = 인덱스
- `arr[key]` = 값

---

# 🔹 객체에서 차이

```jsx
const user = {
  name: 'Tom',
  age: 20
};
```

### ✔ for...in (정상)

```jsx
for (letkeyinuser) {
	console.log(key,user[key]);
}
```

👉 결과

```
name Tom
age 20
```

---
### ❌ for...of (에러)

```jsx
for (let value of user) {
  console.log(value);
}
```

👉 ❗ 에러 발생
👉 객체는 iterable이 아님

---
# 🚨 중요한 포인트

### ❗ 배열에서는 `for...in` 쓰지 않는 게 좋음

이유:

```jsx
Array.prototype.test = 'hello';

const arr = ['a', 'b'];

for (let key in arr) {
  console.log(key);
}
```

👉 결과

```
0
1
test   ❗ 이상한 값 포함됨
```

👉 프로토타입까지 순회됨 😱

---
# ✅ 언제 뭘 써야 하나

|상황|추천|
|---|---|
|배열 값 반복|✅ `for...of`|
|객체 순회|✅ `for...in`|
|인덱스 필요|`for...in` or `forEach`|
|안전한 반복|`for...of`|

---

# String 메소드

## 1. `charAt(index)`

- 지정한 `index` 위치의 문자를 반환. (문자열 인덱스는 0부터 시작)
- 해당 인덱스에 문자가 없으면 빈 문자열(`""`) 반환.

```jsx
let str = "Hello";
console.log(str.charAt(0));  // H
console.log(str.charAt(4));  // o
console.log(str.charAt(10)); // "" (빈 문자열)
```

---
## 2. `indexOf(문자열, [시작위치])`

- 왼쪽(앞) 에서 부터 지정한 문자열을 찾아 **인덱스 번호** 반환.
- 찾지 못하면 -1 반환.
- `시작위치`를 지정하면 그 인덱스부터 검색 시작.

```jsx
let str = "JavaScript";
console.log(str.indexOf("a"));      // 1 (첫 번째 a의 위치)
console.log(str.indexOf("a", 2));   // 3 (인덱스 2부터 검색)
console.log(str.indexOf("z"));      // -1 (없음)
```

---
## 3. `lastIndexOf(문자열, [시작위치])`

- 오른쪽(뒤) 에서 부터 문자열을 찾아 인덱스 반환.
- 찾지 못하면 -1 반환.
- `시작위치`를 지정하면 그 인덱스부터 거꾸로 검색.

```jsx
let str = "banana";
console.log(str.lastIndexOf("a"));     // 5 (마지막 a 위치)
console.log(str.lastIndexOf("a", 4));  // 3 (인덱스 4부터 역방향 검색)
console.log(str.lastIndexOf("z"));     // -1 (없음)
```

---
## 4. 문자열 추출하기

### (1) `substring(index1, index2)`
- `index1`부터 `index2` **직전까지** 문자열 반환.
- `index2`를 생략하면 끝까지 잘라냄.
- `index1`이 `index2`보다 크면 자동으로 위치 교환.

```jsx
let str = "JavaScript";
console.log(str.substring(0, 4));  // "Java" (0~3)
console.log(str.substring(4));     // "Script" (4부터 끝까지)
console.log(str.substring(5, 2));  // "vaS" (인덱스 자동 교환: 2~4)
```

---
### (2) `substr(index, length)`
- `index`부터 `length` 길이만큼 잘라냄.
- `length`를 생략하면 끝까지 반환.
    ⚠️ `substr()`는 **비표준**(deprecated) → 최신 코드에서는 `substring` 또는 `slice` 권장.

```jsx
let str = "JavaScript";
console.log(str.substr(0, 4));  // "Java" (0부터 4글자)
console.log(str.substr(4, 6));  // "Script" (4부터 6글자)
console.log(str.substr(4));     // "Script" (끝까지)
```

---
## 5. `split("구분자")`
- 지정한 구분자를 기준으로 문자열을 나누어 **배열(Array)** 반환.

```jsx
let str = "apple,banana,orange";
let arr = str.split(",");
console.log(arr);  // ["apple", "banana", "orange"]

console.log("2025-09-16".split("-"));
// ["2025", "09", "16"]
```

# ✅ 요약
| 메서드                       | 기능              | 예시                                       |
| ------------------------- | --------------- | ---------------------------------------- |
| `charAt(index)`           | 특정 위치의 문자 반환    | `"Hello".charAt(1)` → `"e"`              |
| `indexOf(str, [pos])`     | 왼쪽에서 찾기         | `"banana".indexOf("a")` → `1`            |
| `lastIndexOf(str, [pos])` | 오른쪽에서 찾기        | `"banana".lastIndexOf("a")` → `5`        |
| `substring(start, end)`   | 구간 추출 (end 전까지) | `"JavaScript".substring(0,4)` → `"Java"` |
| `substr(start, length)`   | 시작점부터 길이만큼 추출   | `"JavaScript".substr(4,6)` → `"Script"`  |
| `split("구분자")`            | 문자열을 배열로 분리     | `"a,b,c".split(",")` → `["a","b","c"]`   |

# DOM 트리 순회(Traversal) 핵심 정리
## 0) Node vs Element 구분
- **Node(노드)**: 요소, 텍스트, 주석 등 트리의 모든 단위.
- **Element(요소)**: 실제 태그(`<div>`, `<li>`…)만.
- 이름에 **Element**가 붙어 있으면 _요소만_ 다룸. (공백 텍스트는 무시)
    - `firstChild` ↔ `firstElementChild`
    - `nextSibling` ↔ `nextElementSibling`
    - `childNodes` ↔ `children`

---
## 1) 부모 찾기
### parentNode / parentElement

```html
<ul id="menu">
  <li class="item">A</li>
</ul>
<script>
  const li = document.querySelector('.item');
  console.log(li.parentNode);     // <ul id="menu">...</ul>
  console.log(li.parentElement);  // <ul id="menu">...</ul> (대부분 같음)
</script>
```

- `parentNode`: 부모가 *문서(document)*이거나 _요소_일 수 있음.
- `parentElement`: 부모가 **요소**일 때만 반환(문서면 `null`)

### closest(selector) ← 강추
- **자기 자신 포함**, 위로 올라가며 셀렉터에 **가장 가까운 조상 요소** 찾기.

```jsx
const btn = document.querySelector('.btn');
const card = btn.closest('.card'); // .card 조상 요소
```

사용 예: 버튼 클릭 시 카드 단위로 처리, 폼 안의 특정 래퍼 찾기 등.

---

## 2) 자식 접근
### children / childNodes / childElementCount / hasChildNodes()

```html
<ul id="menu">
  Text
  <li>A</li>
  <li>B</li>
</ul>

<script>
  const ul = document.getElementById('menu');
  console.log(ul.childNodes);        // NodeList(텍스트, li, … 전부)
  console.log(ul.children);          // HTMLCollection(li, li) ← 요소만
  console.log(ul.childElementCount); // 2
  console.log(ul.hasChildNodes());   // true (텍스트만 있어도 true)
</script>
```

- `childNodes`: **텍스트 노드(공백 포함)** 까지 반환.
- `children`: **요소만** 반환
- `hasChildNodes()`: 텍스트만 있어도 `true`

### firstChild / lastChild vs firstElementChild / lastElementChild

```html
<ul id="menu">
  Text
  <li>A</li>
  <li>B</li>
</ul>

<script>
	const ul = document.getElementById('menu');
 
	console.log( ul.firstChild); // 공백(#text)일 수 있음
	console.log(ul.firstElementChild); // <li>A</li>
	console.log(ul.lastChild); // 공백(#text)일 수 있음
	console.log(ul.lastElementChild); // <li>B</li>
 </script>
```

- _실무에서는 대부분 Element 버전_을 선호.

---
## 3) 형제 접근

### previousSibling / nextSibling vs previousElementSibling / nextElementSibling

```html
<div id="wrap">
  <h2>Title</h2>
  <!-- comment -->
  <p class="a">A</p>
  <p class="b">B</p>
</div>

<script>
  const b = document.querySelector('.b');

  console.log(b.previousSibling);         // 공백/주석 등 #text/#comment 가능
  console.log(b.previousElementSibling);  // <p class="a">A</p>  ← 요소만

  console.log(b.nextSibling);             // 공백/텍스트 가능
  console.log(b.nextElementSibling);      // 다음 요소(없으면 null)
</script>
```

- 공백/주석에 걸려 당황하는 일이 많음 → **Element 버전 사용 습관화**.

---
## 4) 접근·검색 보조 메서드(함께 쓰면 편함)

- `querySelector(selector)` / `querySelectorAll(selector)`

- `matches(selector)`: 요소가 셀렉터에 부합하는지
    - **역할**: 해당 요소가 특정 CSS 선택자(selector)와 일치하는지 확인.
    - **반환값**: `true` 또는 `false`.
    - **용도**: “이 요소가 내가 원하는 조건을 만족하는가?”를 빠르게 검사.

- `contains(node)`: 어떤 노드를 자손으로 포함하는지
    - **역할**: 한 노드가 다른 노드를 자식(후손 포함)으로 가지고 있는지 검사.
    - **반환값**: `true` 또는 `false`.
    - **특징**
        - 자기 자신을 인자로 넣으면 `true` 반환.
        - 요소뿐 아니라 **텍스트 노드**에도 적용 가능.

```jsx
<ul class="list">
  <li class="item selected">첫 번째</li>
  <li class="item">두 번째</li>
</ul>

<script>
 const item = document.querySelector('.item'); // 첫 번째 li
 const list = document.querySelector('.list');

 console.log(item.matches('.item.selected')); // true
 console.log(item.matches('.item.active'));   // false

 console.log(list.contains(item));            // true
 console.log(item.contains(list));            // false
</script>
```

## 5) 자주 쓰는 조작 메서드(순회와 함께)
- **삽입/이동**:
    - `append`, `prepend` - 하위(자식) (자식 노드 추가)
    - `before`, `after` - 형제 기준으로 추가 (형제 노드 추가)
    - `appendChild`, `insertBefore`
- **삭제**: `remove`, `removeChild`
- **교체**: `replaceWith`, `replaceChild`

```html
<ul class="list">
  <li class="item selected">첫 번째</li>
  <li class="item">두 번째</li>
</ul>

<script>
const li = document.createElement('li');
li.textContent = 'NEW'; // <li>NEW</li>

const ul = document.querySelector('ul');
ul.append(li);               // 마지막 자식으로

ul.firstElementChild.before(li); // 첫 자식 앞(이동됨)

li.remove();                 // 제거

</script>
```

- `div.innerHTML = "<h1>안녕</h1>` -> 덮어쓰기, 이어쓰기

## 6) 예제

```html
<ul id="menu">
  <!-- 공백도 노드임 -->
  <li class="item a">A</li>
  <li class="item b">B</li>
  <li class="item c active">C</li>
</ul>

<script>
  const ul = document.getElementById('menu');
  const c = ul.querySelector('.c');

  // 부모
  console.log(c.parentElement === ul); // true
  console.log(c.closest('#menu'));     // <ul id="menu">

  // 형제 (요소만)
  console.log(c.previousElementSibling.className); // "item b"
  console.log(c.nextElementSibling);               // null

  // 자식 
  console.log(ul.children.length);     // 3
  console.log(ul.firstElementChild.textContent); // "A"

</script>
```

# `data-*` **사용자 정의 속성(Custom Data Attribute)**

## 정의

- `data-이름="값"` 형태로 작성.
- 표준 HTML 속성이 아니라 **개발자가 임의로 붙이는 속성**이지만, HTML5 명세에 정식으로 포함되어 있다.
- 반드시 접두사 `data-`로 시작해야 하고, 그 뒤에는 원하는 이름을 붙일 수 있다. (`data-user-id`, `data-action`, `data-color` 등)

---
## ✅ 언제 쓰나?

1. **DOM 요소에 관련된 추가 정보 저장**
    
    - HTML에 표시할 필요는 없지만, 스크립트가 활용할 수 있는 데이터를 저장할 때.
2. **JavaScript와의 연결고리**
    
    - 버튼을 눌렀을 때 어떤 작업을 할지, 어떤 ID를 참고해야 할지 알려줄 때.
3. **스타일링과 구분(보조용)**
    
    - CSS 선택자로도 접근 가능 (`[data-state="active"] { … }`).
    
    `data-속성`은 **작고 간단한 데이터**를 담을 때가 제일 좋다.
    

---
## ✅ 사용 예시

### 1) HTML에 데이터 저장

```html
<button data-user-id="101" data-action="delete">삭제</button>
<button data-user-id="102" data-action="update">수정</button>
```

### 2) JavaScript에서 읽기/쓰기

```jsx
const btn = document.querySelector('[data-action="delete"]');

// data 속성 읽기 - dataset 사용
console.log(btn.dataset.userId);  // "101"
console.log(btn.dataset.action);  // "delete"

//data 속성 읽기 - getAttribute 함수 사용
console.log(btn.getAttribute("data-user-id"));  // "101"
console.log(btn.getAttribute("data-action"));  // "delete"

// data 속성 쓰기
btn.dataset.action = "disabled";
console.log(btn.dataset.action);  // "disabled"
```

👉 `dataset` 객체를 통해 `data-속성` 값에 접근

👉 속성 이름은 **카멜케이스**로 바뀐다. (`data-user-id` → `dataset.userId`)

---
### 3) CSS에서 활용

```css
button[data-action="delete"] {
  background-color: red;
  color: white;
}
```

---
## ✅ 요약

- **정의**: HTML 요소에 추가 정보를 저장할 수 있는 사용자 정의 속성.
- **형태**: `data-이름="값"`
- **사용처**: JS에서 간단한 데이터 참조, UI 상태값 저장, 특정 동작 구분.
- **접근법**: `element.dataset.이름`

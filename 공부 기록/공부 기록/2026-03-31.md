* 동기(실행하는 동안 다른 작업을 동시에 실행 못 함)
* 비동기(실행하는 동안 다른 작업을 동시에 실행 가능)
	* `setInterval()`

# 구조분해 할당 (**Destructuring** Assignment)

## 1. 구조분해 할당이란?

- **배열(Array)이나 객체(Object)의 값을 분해하여 변수에 쉽게 할당하는 문법**
- ES6(ECMAScript 2015)에서 도입
- 복잡한 데이터 구조에서 필요한 값을 간단히 꺼내 쓸 수 있음

---
## 2. 왜 사용할까? (필요성)

✅ 코드 가독성 향상 → 변수 할당이 짧고 명확해짐

✅ 중복 코드 줄이기 → 같은 값을 여러 번 접근할 필요 없음

✅ 함수 반환 값 처리 편리 → 배열·객체 반환 시 원하는 값만 바로 꺼내기 가능

---
## 3. 사용 방법

### (1) 배열 구조 분해 할당

```jsx
const colors = ["red", "green", "blue"];

// 기존 방식
const first = colors[0];
const second = colors[1];

// 구조분해 할당
const [c1, c2, c3] = colors;

console.log(c1); // red
console.log(c2); // green
console.log(c3); // blue
```

➡ 순서(index)에 따라 변수에 값이 들어감

### 일부 값만 할당

```JavaScript
const colors = ["red", "green", "blue"];

const [first, , third] = colors;
console.log(first, third); // red blue

```

### 기본값 지정

```jsx
const [a, b, c = "default"] = ["one", "two"];
console.log(a, b, c); // one two default
```

---

### (2) 객체 구조분해 할당

```jsx
const user = { name: "Alice", age: 25 };

// 기존 방식
const userName = user.name;
const userAge = user.age;

// 구조분해 할당
const { name, age } = user;
console.log(name, age); // Alice 25

```

➡ 키(key) 이름을 기준으로 변수에 값이 들어감

### 변수 이름 바꾸기 (별칭 사용)

```jsx
const user = { name: "Alice", age: 25 };

const { name: userName, age: userAge } = user;
console.log(userName, userAge); // Alice 25

```

### 기본값 지정

```jsx
const user = { name: "Alice", age: 25 };
const { name, age , job = "student" } = user;

console.log(name);// Alice
console.log(age); // 25
console.log(job); // student

// ------------------------------------------------------
const user = { name: "Alice", age: 25 ,job:"teacher"};
const { name, age , job = "student" } = user;

console.log(name);// Alice
console.log(age); // 25
console.log(job); // teacher
```

---

### (3) 함수와 함께 사용하기

### 함수 반환값 구조분해

```jsx
function getPoint() {
  return [10, 20];
}

// 기존 방식
const arr = getPoint();
console.log(arr); // [10,20]

//구 조분해
const [x, y] = getPoint();
console.log(x, y); // 10 20
```

### 매개변수 구조분해

```jsx
// 기존 방식
 function introduce(obj) {
  console.log(`이름: ${obj.name}, 나이: ${obj.age}`);
}

introduce({ name: "Bob", age: 30 });
// 이름: Bob, 나이: 30

//---------------------------------------------------
// 구조분해
function introduce({ name, age }) {
  console.log(`이름: ${name}, 나이: ${age}`);
}

introduce({ name: "Bob", age: 30 });
// 이름: Bob, 나이: 30
```

---

## 4. 정리

- **배열** → 순서(index) 기준으로 값 추출
- **객체** → key 이름 기준으로 값 추출
- **장점**
    - 코드 간결, 가독성 향상
    - 함수 반환 값, 매개변수 활용에 유용
- **활용 예시**: React props, API 데이터 처리, 함수 반환 값 등

# Spread 연산자 (`...`)

## 1. Spread 연산자란?

- **배열(Array)이나 객체(Object)의 요소를 개별 값으로 펼쳐주는 연산자**
- 기호: `...` (점 3개)
- ES6(ECMAScript 2015)에서 도입
- 배열, 객체, 함수 호출 시 다양하게 활용 가능

---
## 2. 왜 사용할까? (필요성)

✅ 배열/객체의 **복사 및 병합**을 간단하게 할 수 있음

✅ **불변성(immutability)** 유지 → **원본을 해치지 않고 새 값 생성**

✅ 함수 호출 시 배열을 인자로 쉽게 전달 가능

✅ 데이터 전개 시 코드 가독성 및 간결성 향상

---
## 3. 사용 방법

### (1) 배열에서 사용

### 배열 복사

```jsx
const arr1 = [1, 2, 3];
const arr2 = [...arr1];

console.log(arr2); // [1, 2, 3]
console.log(arr1 === arr2); // false (서로 다른 배열)
```

### 배열 합치기

```jsx
const a = [1, 2];
const b = [3, 4];
const c = [...a, ...b];

console.log(c); // [1, 2, 3, 4]
```

### 요소 추가하기

```jsx
const numbers = [2, 3, 4];
const newNumbers = [1, ...numbers, 5];
console.log(newNumbers); // [1, 2, 3, 4, 5]
```

---
### (2) 객체에서 사용

### 객체 복사

```jsx
const user = { name: "Alice", age: 25 };
const copyUser = { ...user };

console.log(copyUser); // { name: "Alice", age: 25 }
console.log(user === copyUser); // false
```

### 객체 병합

```jsx
const base = { name: "Alice", age: 25 };
const extra = { job: "Developer", age: 30 };

const merged = { ...base, ...extra };
console.log(merged);
// { name: "Alice", age: 30, job: "Developer" }
// 뒤에 오는 값이 앞의 값을 덮어씀
```

---
### (3) 함수 호출 시 사용

### 배열을 인자로 전달

```jsx
function sum(x, y, z) {
  return x + y + z;
}

const nums = [1, 2, 3];

console.log(sum(...nums)); // 6
```

➡ `sum(nums)`와 달리, 배열 요소가 각각의 매개변수에 들어감

---
### (4) 문자열 전개

```jsx
const str = "hello";
const letters = [...str];

console.log(letters); // ["h", "e", "l", "l", "o"]
```

---
## 4. 정리

- **정의**: 배열/객체/문자열을 개별 값으로 펼쳐주는 연산자
- **왜 쓰나**: 복사, 병합, 함수 호출 시 간편 & 불변성 유지
- **사용처**
    - 배열 복사/합치기
    - 객체 병합
    - 함수 인자 전달
    - 문자열 분해

# Optional Chaining (`?.`)

## 1. Optional Chaining이란?

- **객체의 속성이나 메서드에 안전하게 접근하기 위한 문법**
- 중첩된 객체 구조에서 **존재하지 않는 속성을 접근해도 오류(`TypeError`)가 발생하지 않고 `undefined`를 반환**
- 문법: `obj?.prop`, `obj?.[expr]`, `obj?.method?.()`
- ES2020(ES11)에서 도입

---

## 2. 왜 사용할까? (필요성)

✅ **안전성** → 존재하지 않는 속성을 접근할 때 에러 방지

✅ **가독성** → 복잡한 `if문`이나 `&&` 연산자 대신 간단히 표현

✅ **실무 활용** → API 응답 데이터, 사용자 입력 등 불확실한 값 처리에 유용

---
## 3. 사용 방법

### (1) 기본 객체 접근

```jsx
const user = { name: "Alice", 
               profile: { age: 25 }
              };

// 기존 방식
console.log(user && user.profile && user.profile.age); // 25

// Optional Chaining
console.log(user?.profile?.age); // 25
```

➡ `user.profile`이 존재하지 않으면 `undefined` 반환

---

### (2) 존재하지 않는 속성 접근

```jsx
const user = { name: "Bob" };

console.log(user.profile?.age);
// undefined (에러 발생 ❌)

```

---

### (3) 함수 호출

```jsx
const obj = {
  greet() {
    return "Hello!";
  }
};

console.log(obj.greet?.());   // Hello!
console.log(obj.sayHi?.());   // undefined (에러 발생 ❌)
```

---

### (4) 배열/인덱스 접근

```jsx
const arr = [10, 20, 30];

console.log(arr?.[1]);  // 20
console.log(arr?.[5]);  // undefined
```

---
## 4. 정리

- **정의**: 존재하지 않는 객체 속성/메서드 접근 시 `undefined`를 반환하는 안전한 연산자
- **장점**: 에러 방지, 코드 간결화, 가독성 향상
- **사용처**
    - 중첩 객체 접근 (API 응답 데이터 처리)
    - 함수/메서드가 존재할 때만 실행
    - 배열 요소 접근

# Nullish 병합 연산자 (`??`)

## 1. Nullish 병합 연산자란?

- **`null` 또는 `undefined`일 때만 대체 값을 반환하는 연산자**
- 문법: `값1 ?? 값2`
- `값1`이 `null` 또는 `undefined` → `값2` 반환
- 그렇지 않으면 `값1` 반환
- ES2020(ES11)에서 도입

---
## 2. 왜 사용할까? (필요성)

✅ **기본값 설정**을 안전하게 하기 위해

✅ `0`, `""`, `false` 같은 **falsy 값**과 구분 가능

✅ 기존의 `||`(OR) 연산자보다 **의도한 기본값 처리**를 명확히 할 수 있음

---

## 3. 사용 방법

### (1) 기본 예시

```jsx
let name = null;
let user = name ?? "Guest";

console.log(user); // Guest
```

➡ `name`이 `null`이므로 `"Guest"` 반환

---
### (2) `||`와의 차이

```jsx
let count = 0;

// OR (||) 사용
console.log(count || 10); // 10 (0도 falsy로 처리)

// Nullish (??) 사용
console.log(count ?? 10); // 0 (null/undefined만 체크)
```

➡ `??`는 `null`과 `undefined`만 "없다"고 간주

➡ `0`, `""`, `false`는 **정상적인 값으로 인정**

---

### (3) 객체 속성에서 기본값 설정

```jsx
const user = { name: "Alice", age: null };

console.log(user.age ?? 18); // 18 (age가 null이므로 기본값 반환)
```

---

### (4) 함수 매개변수에서 활용

```jsx
function greet(name) {
  let userName = name ?? "Guest";
  console.log(`Hello, ${userName}`);
}

greet("Bob");   // Hello, Bob
greet(null);    // Hello, Guest
greet();        // Hello, Guest
```

---

## 4. 정리

- **정의**: `null` 또는 `undefined`일 때만 기본값을 적용하는 연산자
- **장점**
    - 불필요한 `if`문 줄임
    - `||` 연산자보다 직관적 (**특히 `0`, `false`, `""`를 값으로 인정할 때**)
- **활용처**
    - 변수 초기값
    - 객체 속성 기본값
    - 함수 매개변수 처리

# 클로저(Closure)

## 1. 클로저(Closure)란?

- **정의**:
	- 클로저는 **함수와 그 함수가 선언될 당시의 렉시컬 환경(Lexical Environment)의 조합**을 말한다.
	- 쉽게 말하면, **함수가 자신이 선언된 스코프(환경)를 기억하고, 스코프 밖에서 호출되더라도 그 환경에 접근할 수 있는 것**이다.

💡 **렉시컬 환경(Lexical Environment):**
>현재 실행 중인 코드 블록의 식별자 정보를 관리합니다. 즉, 현재 실행 컨텍스트에서 사용되는 변수와 함수를 담고 있으며, 이를 통해 스코프와 식별자의 조회 및 변경 등이 수행됩니다.

---
## 2. 클로저의 기본 예시

```jsx
function outer() {
  let count = 0;   // outer의 지역변수

  function inner() {
    count++;
    console.log(count);
  }

  return inner;
}

const counter = outer();  // inner 함수가 반환됨
counter(); // 1
counter(); // 2
counter(); // 3
```

👉 여기서 중요한 점:
- `outer()` 실행이 끝났는데도 `counter`는 `count` 변수에 접근 가능.
- 이유는 `inner` 함수가 `count`를 포함하는 ==**환경을 기억**==하고 있기 때문 → 이것이 **클로저**!

---

## 3. 클로저가 왜 필요한가?

1. ⭐**데이터 은닉 (캡슐화)**
    - 외부에서 직접 접근하지 못하게 하고, 함수로만 접근 가능하게 함.
```jsx
    function makeCounter() {
      let count = 0;
      return {
        increase: () => ++count,
        decrease: () => --count,
        value: () => count
      };
    }
    
    const counter = makeCounter();
    
    console.log(counter.increase()); // 1
    console.log(counter.increase()); // 2
    console.log(counter.decrease()); // 1
    console.log(counter.value());    // 1
```

1. **상태 유지**
    - 함수 실행이 끝나도 변수가 메모리에 유지됨.
    - 비동기 콜백에서 특정 값을 “기억”하도록 할 때 유용.

2. **모듈 패턴 구현**
    - 클로저를 사용해 외부에 노출하고 싶은 것만 리턴 → 정보 은닉 및 구조화.

---

## 4. 클로저 사용 예시

- **이벤트 핸들러에서 상태 유지**
```jsx
    function makeButtonHandler() {
      let clicked = 0;
      return function() {
        clicked++;
        console.log(`버튼이 ${clicked}번 눌렸습니다`);
      };
    }
    
    const handler = makeButtonHandler();
    document.getElementById("btn").addEventListener("click", handler);
```

- **타이머에서 값 캡처**
```jsx
for (let i = 1; i <= 3; i++) {
      setTimeout(() => {
        console.log(i);  // i의 값이 유지됨 (1, 2, 3)
      }, i * 1000);
	}
```

---
## 5. 주의할 점

- **메모리 누수 위험**: 클로저는 참조를 유지하므로 불필요한 데이터가 계속 메모리에 남을 수 있음.
- **남용 주의**: 모든 곳에 쓰면 코드 가독성이 떨어질 수 있음.

---

✅ **정리**
> 클로저는 “**함수가 자신이 선언될 당시의 스코프를 기억해서, 외부 함수 실행이 끝난 뒤에도 그 스코프에 접근할 수 있게 하는 기능**”입니다.

→ **데이터 은닉, 상태 유지, 모듈화**에 활용할 수 있는 강력한 개념

# JavaScript의 `this` 키워드

## 1. `this`란 무엇인가?

- **현재 실행 중인 컨텍스트(문맥)에서 자신을 가리키는 참조 변수**
- `this`는 **값이 고정되지 않고, 실행되는 방식(호출 방식)에 따라 달라짐**
- 즉, **“누가 호출했는지”** 에 따라 `this`가 달라진다.

---
## 2. 상황별 `this`의 의미

### (1) 전역 컨텍스트(Global Context)

```jsx
console.log(this);
```

- 브라우저: `window` 체

---
### (2) 함수 호출

```jsx
function showThis() {
  console.log(this);
}

showThis();// 브라우저: window

```

➡ 일반 함수 호출에서는 전역 객체를 가리킴

---
### (3) 메서드 호출 (객체 안의 함수)

```jsx
const user = {
  name: "Alice",
  greet() {
    console.log(this.name);
  }
};

user.greet(); // Alice
```

➡ 메서드를 호출한 **객체 자신(user)** 을 가리킴

---
### (4) 메서드 분리해서 호출

```jsx
const user = {
  name: "Alice",
  greet() {
    console.log(this.name); // this: window 함수
  }
};

const fn = user.greet;
fn();
// window.name (브라우저 전역) 또는 undefined
```

👉 **주의**: 메서드를 변수에 할당하면 원래 객체와의 연결이 끊겨서 `this`가 달라짐

---
### (5) 화살표 함수

```jsx
const obj = {
  name: "Alice",
  greet: () => {
    console.log(this); // window
    console.log(this.name); // 공백
  }
};

obj.greet(); // 

// ---------------------------------------------------
const obj = {
  name: "Alice",
  greet: function() {
    console.log(this); // {name: 'Alice', greet: ƒ}
    console.log(this.name); // Alice
  }
};

obj.greet(); //
```

➡ 화살표 함수는 자신만의 `this` 를 갖지 않고, **상위 스코프의 `this`** 를 그대로 사용

### 예시 (실무에서 유용)

```jsx
const user = {
  name: "Alice",
  show: function() {
    setTimeout(() => {
      console.log(this); // {name: 'Alice', show: ƒ}
      console.log(this.name); // Alice
    }, 1000);
  }
};

user.show(); // Alice
```

👉 `setTimeout` 안에서 화살표 함수를 쓰면 상위 `user`의 `this`를 유지

---
### (7) 이벤트 핸들러

```jsx
const btn = document.querySelector("button");

btn.addEventListener("click", function() {
  console.log(this); // 클릭된 버튼
});

btn.addEventListener("click", () => {
  console.log(this); // window (상위 스코프의 this)
});
```

👉 **주의**: 이벤트 핸들러에서 화살표 함수를 쓰면 `this`가 버튼이 아닌 전역을 가리킴

---
## 3. `this` 정리

|상황|`this`가 가리키는 것|
|---|---|
|전역|브라우저: `window`|
|일반 함수 호출|전역 객체 (strict 모드: `undefined`)|
|객체의 메서드|해당 객체|
|화살표 함수|자신만의 `this` 없음 → 상위 스코프의 `this`|
|이벤트 핸들러|일반 함수: 이벤트 대상, 화살표 함수: 상위 스코프|

---
## 4. 주의해야 할 점

- `this`는 **정의될 때가 아니라, ==호출될 때 결정**된다.==
- 화살표 함수는 `this`를 바인딩하지 않으므로, 메서드 정의에는 잘 쓰지 않는다.
- 메서드를 다른 변수에 할당해서 호출하면 원래 객체와의 연결이 끊긴다.
- 이벤트 핸들러에서 화살표 함수를 쓰면 원하는 `this`를 얻지 못 할 수 있다.

# HTML 웹 스토리지 API
> HTML 웹 스토리지 API; 쿠키보다 낫습니다.
## HTML 웹 스토리지란 무엇인가요?

- 웹 스토리지를 사용하면 애플리케이션이 사용자의 브라우저 내에 데이터를 로컬로 저장할 수 있습니다.

- HTML5 이전에는 애플리케이션 데이터를 모든 서버 요청에 포함되는 쿠키에 저장해야 했습니다. 웹 스토리지는 더 안전하며, 웹사이트 성능에 영향을 주지 않고 대량의 데이터를 로컬에 저장할 수 있습니다.

- 쿠키와 달리 저장 용량 제한이 훨씬 크고(최소 5MB) 정보가 서버로 전송되는 일은 절대 없습니다.

- 웹 저장소는 Origin별(도메인 및 프로토콜별)로 관리됩니다. 동일한 Origin의 모든 페이지는 동일한 데이터를 저장하고 접근할 수 있습니다.


---
## 웹 스토리지 API 객체
> 웹 스토리지는 브라우저에 데이터를 저장하기 위한 두 가지 객체를 제공합니다.

- `window.localStorage`- 만료 기한 없이 데이터를 저장합니다 (브라우저 탭을 닫아도 데이터가 손실되지 않습니다).
- `window.sessionStorage`- 한 세션 동안 데이터를 저장합니다 (브라우저 탭을 닫으면 데이터가 손실됩니다).

```jsx
<script>  
const x = document.getElementById("result");  
if (typeof(Storage) !== "undefined") {  
  x.innerHTML = "Your browser supports Web storage!";  
} else {  
  x.innerHTML = "Sorry, no Web storage support!";  
}  
</script>
```

## localStorage 객체
> 해당 `localStorage`객체는 만료일 없이 데이터를 저장합니다. 브라우저를 닫더라도 데이터는 손실되지 않으며, 다음 날, 다음 주 또는 다음 해에도 사용할 수 있습니다.

### 예
`localStorage`이름과 값 쌍을 설정하고 검색하는 데 사용합니다 .

```jsx
<script>  
const x = document.getElementById("result");  
  
if (typeof(Storage) !== "undefined") {  
  // Store  
  localStorage.setItem("lastname", "Smith");  
  localStorage.setItem("bgcolor", "yellow");  
  // Retrieve  
  x.innerHTML = localStorage.getItem("lastname");  
  x.style.backgroundColor = localStorage.getItem("bgcolor");  
} else {  
  x.innerHTML = "Sorry, no Web storage support!";  
}  
</script>
```

**예시 설명:**
- 이 `localStorage.setItem()`메서드를 사용하여 이름/값 쌍을 생성합니다.
- 해당 메서드를 사용하여 `localStorage.getItem()`설정된 값을 검색합니다.
- "lastname" 값을 가져와서 id가 "result"인 요소에 삽입합니다.
- "bgcolor" 값을 가져와서 id가 "result"인 요소의 backgroundColor 스타일 속성에 삽입합니다.

✔ **"lastname" 로컬 스토리지 항목을 제거하는 구문은 다음과 같습니다.**
```jsx
localStorage.removeItem("lastname");
```

**참고:** 이름/값 쌍은 항상 문자열로 저장됩니다. 필요할 때 다른 형식으로 변환하는 것을 잊지 마세요!
## sessionStorage 객체
> 해당 `sessionStorage`객체는 기존 객체와 동일 **하지만**`localStorage` , 단 하나의 세션 동안만 데이터를 저장한다는 점이 다릅니다! 사용자가 특정 브라우저 탭을 닫으면 데이터가 삭제됩니다.

### sessionStorage를 사용한 클릭 수 계산

다음 예시는 현재 세션에서 사용자가 버튼을 클릭한 횟수를 계산합니다
### 예
```jsx
<script>  
function clickCounter() {  
  const x = document.getElementById("result");  
  if (typeof(Storage) !== "undefined") {  
    if (sessionStorage.clickcount) {  
      sessionStorage.clickcount = Number(sessionStorage.clickcount)+1;  
    } else {  
      sessionStorage.clickcount = 1;  
    }  
    x.innerHTML = "You have clicked the button " + sessionStorage.clickcount + " time(s) in this session!";  
  } else {  
    x.innerHTML = "Sorry, no Web storage support!";  
  }  
}  
</script>
```

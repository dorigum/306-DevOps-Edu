# 제네릭(Gemeric)
>클래스나 메소드....

### 왜 사용하는가?
#### 1. 타입 안정성(Type Safety)
: 컴파일 시점에 타입 체크 가능
#### 2. 형변환 제거
: 불필요한 캐스팅 제거
#### 3. 재사용성 증가
: 하나의 코드로 여러 타입 처리 가능

### 제네릭이 없던 시절의 문제점

```java
class Box {
	private Object data;
	
	public void set(Object data) {
		this.data = data;
	}
	
	public Object get() {
		return data;
	}
 }
```


#### 👉

### 제네릭 사용 후
```java
Box<>
```

### 우리가 알아야 하는 제네릭의 종류
```
1. 제네릭 타입(Generic Type)
2. 타입 파라미터
3. 제네릭 메소드
4. 와일드 카드
```

### 제네릭 타입(Generic Type)
#### 정의
> 클래스 또는 인터페이스에 타입 파라미터를 선언하는 것

```java
class Box<T> { }
```

# 포장 클래스
### 포장 객체
- 기본 타입
- 포장하고 있는 
```java
int i = 5;
Integer it = new Integer(i); // 기본형을 객체로 변환

int no = it.getValue(); // 객체를 기본형으로 변환
// --------------------------------

```

### 박싱과 언박싱
- **AutoBoxing**
	- 기본형을 객체로 변환
```java
int i = 5;
Integer it = i;
```

- **UnBoxing**
	- 객체를 기본형으로 변환
```java
int no = it;
```

# 제네릭 메소드
- 타입 파라미터를 가지고 있는 메소드
- 타입 파라미터가 메소드 선언부에 정의
- 리턴 타입 앞에 `<>` 기호를 추가하고 타입 파라미터 정의 후, 리턴 타입과 매개변수 타입에서 사용
```java
public <A, B, ...> 리턴타입 메소드명(매개변수, ...) {...}
// <A, B, ...> -> 타입 파라미터 정의
```

## 와일드카드 타입 파라미터
- 제네릭 타입을 매개값이나 리턴 타입으로 사용할 때, 범위에 있는 모든 타입으로 대체할 수 있는 타입 파라미터
- `?` 로 표시
```java
리턴타입 메소드명(제네릭타입<? extends Student> 변수) {...}
리턴타입 메소드명(제네릭타입<? super Worker> 변수) {...}
리턴타입 메소드명(제네릭타입<?> 변수) {...}
```


## 람다식
- 함수형 인터페이스(메소드가 하나일 때) 에서만 사용 가능
- **함수형 프로그래밍**
	- 함수를 정의하고 이 함수를 데이터 처리부로 보내서 데이터를 처리하는 기법
- 데이터 처리부는 제공된 함수의 입력값으로 데이터를 넣고, 함수에 정의된 처리 내용을 실행
- 람다식
	- 데이터 처리부에 제공되는 함수 역할을 하는 매개변수를 가진 중괄호 블록
- Java는 람다식을 익명 구현 객체로 변환


## 함수형 인터페이스
- *인터페이스가 단 하나의 추상 메소드를 가지는 것*
- 인터페이스가 함수형 인터페이스임을 보장하기 위해서는 `@FunctionalInterface` 어노테이션을 붙임
- `@FunctionalInterface`
	- 컴파일 과정에서 추상 메소드가 하나인지 검사해서, 정확한 함수형 인터페이스를 작성할 수 있게 도와주는 역할
```java
// 인터페이스
public interface Runnable {
	void run();
}

// 람다식

```

# 컬렉션 자료구조
## 컬렉션 프레임워크
- 널리 알려진 자료 구조를 바탕으로 객체들을 효율적으로 추가, 삭제, 검색할 수 있도록 관련 인터페이스와 클래스들을 포함시켜 놓은 `java.util` 패키지

- ==주요 인터페이스==
	- List: 중복 가능, 순서를 유지하고 저장
		- ArrayList, Vector, LinkedList
	- Set: 중복 저장 X, 순서를 유지하지 않고 저장
		- HashSet, TreeSet
	- Map: <키, 값>으로 구성된 엔트리 저장, 키는 중복 저장 X
		- HashMap, HashTable, TreeMap, Properties

### List 컬렉션
- 객체를 인덱스로 관리하기 때문에 객체를 저장하면 인덱스가 부여되고 인덱스로 객체를 검색, 삭제할 수 있는 기능을 제공

# Comparable vs Comparator
- Comparable과 Comparator는 모두 인터페이스(interface)
- Comparable 혹은 Comparator를 사용할 때 인터페이스 내에 선언된 메소드는 반드시 재정의 필수
- 객체를 비교할 때 사용한다.
- Comparator는 두 매개변수 객체를 비교한다.
- Comparable은 lang 패키지에 있기 때문에 import를 해줄 필요가 없지만, ==Comparator는 util 패키지에 있기 때문에 반드시 import 필수==

# `data-*` **사용자 정의 속성(Custom Data Attribute)**

## 정의
- `data-이름="값"` 형태로 작성.
- 표준 HTML 속성이 아니라 **개발자가 임의로 붙이는 속성**이지만, HTML5 명세에 정식으로 포함되어 있다.
- 반드시 접두사 `data-`로 시작해야 하고, 그 뒤에는 원하는 이름을 붙일 수 있다. (`data-user-id`, `data-action`, `data-color` 등)

---
## ✅ 언제 쓰나?

1. **DOM 요소에 관련된 추가 정보 저장**
    - HTML에 표시할 필요는 없지만, 스크립트가 활용할 수 있는 데이터를 저장할 때

2. **JavaScript와의 연결고리**
    - 버튼을 눌렀을 때 어떤 작업을 할지, 어떤 ID를 참고해야 할지 알려줄 때

3. **스타일링과 구분(보조용)**
    - CSS 선택자로도 접근 가능 (`[data-state="active"] { … }`).

    `data-속성`은 **작고 간단한 데이터**를 담을 때가 제일 좋다.

---
## ✅ 사용 예시

### 1) HTML에 데이터 저장

```html
<button data-user-id="101" data-action="delete">삭제</button>
<button data-user-id="102" data-action="update">수정</button>
```

### 2) JavaScript에서 읽기/쓰기

```jsx

const btn = document.querySelector('[data-action="delete"]');

// data 속성 읽기 - dataset 사용
console.log(btn.dataset.userId); // "101"
console.log(btn.dataset.action); // "delete"

//data 속성 읽기 - getAttribute 함수 사용
console.log(btn.getAttribute("data-user-id")); // "101"
console.log(btn.getAttribute("data-action")); // "delete"

// data 속성 쓰기
btn.dataset.action = "disabled";
console.log(btn.dataset.action); // "disabled"
```

👉 `dataset` 객체를 통해 `data-속성` 값에 접근

👉 속성 이름은 **카멜 케이스**로 바뀐다. (`data-user-id` → `dataset.userId`)

---
### 3) CSS에서 활용

```css
button[data-action="delete"] {
  background-color: red;
  color: white;
}
```

---
## ✅ 요약

- **정의**: HTML 요소에 추가 정보를 저장할 수 있는 사용자 정의 속성.
- **형태**: `data-이름="값"`
- **사용처**: JS에서 간단한 데이터 참조, UI 상태값 저장, 특정 동작 구분.
- **접근법**: `element.dataset.이름`

# 동적으로 생성된 element에 이벤트 적용

### “이벤트를 언제, 어디에 달았느냐” 중요

#### 삭제 버튼이 이벤트에 의해 동적으로 생성되었고 그 삭제 버튼에 이벤트를 적용하고 싶은 경우

#### 왜 아래 코드는 안될까?

```jsx
document.querySelectorAll('[value=삭제]').forEach(item => {
  item.addEventListener('click', () => console.log('되니???'));
});
```

- `querySelectorAll`은 **코드를 실행한 “그 시점”에 DOM에 존재하는 요소들만** 찾아서 **정적인(NodeList snapshot)** 목록을 만든다.
- 이후에 **동적으로 새로 추가된 버튼**들은 이 `forEach`에 포함되지 않았기 때문에, 그 버튼들에는 핸들러가 **전혀 붙지 않는다.**
- 그래서 “동적으로 생성된 버튼” 에는 클릭이 안 먹는다.
    _(이미 존재하던 버튼이라면 이 코드도 잘 동작한다)_

#### 왜 아래 코드는 잘될까? (이벤트 위임)

```jsx
document.body.addEventListener('click', function(event) {
  if (event.target.matches('[value=삭제]')) {
    let tr = event.target.parentElement.parentElement; // button -> td -> tr
    tr.parentElement.removeChild(tr);
  } // - body에 이벤트 등록 -> 하위로 이벤트가 전이됨
});
```

- 클릭을 **공통 조상(여기선 `body`)** 에 한 번만 달아두고, 실제 클릭된 곳(`event.target`)이 우리가 원하는 버튼인지 **런타임에 검사한다.**
- 클릭 이벤트는 **버블링** 되므로, 나중에 동적으로 추가된 버튼을 클릭해도 그 이벤트가 `body`까지 올라오고, 조건(`matches`)이 맞으면 핸들러가 실행된다.
- 이를 **이벤트 위임(Event Delegation)** 이라고 한다. 동적 요소에 특히 유리하다.

#### 다른 예시)

`event.target`이 꼭 버튼일 필요는 없다 (예: 버튼 안의 `<span>`을 클릭하면 `target`이 `span`이 됨)
그래서 `closest()`로 버튼을 찾는다.

```jsx
	document.addEventListener('click', (e) => {
		const btn = e.target.closest('[value=삭제]');
		if (!btn) return;              // 버튼이 아니면 무시
		const tr = btn.closest('tr');  // 가장 가까운 행 찾기
		tr?.remove();
	});
```

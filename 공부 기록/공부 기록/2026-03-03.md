##### UNIQUE(유니크)
- 중복 값을 허용하지 않음
- 단, NULL 값은 허용함(PRIMARY KEY와의 차이점)
```MYSQL
CREATE TABLE customers (
	id INT PRIMARY KEY AUTO_INCREMENT,
	email VARCHAR(100) UNIQUE -- 이메일은 중복 불가
);
```
- email 컬럼에 중복된 값이 들어가면 오류 발생

##### DEFAULT(기본값)
- 컬럼에 값이 입력되지 않았을 때 자동으로 설정할 **기본값**
```MYSQL
CREATE TABLE products (
	id INT PRIMARY KEY AUTO_INCREMENT,
	name VARCHAR(50) NOT NULL,
	stock INT DEFAULT 10 -- 기본 재고는 10
);
```
- stock 값을 입력하지 않으면 값은 자동으로 10이 설정됨

##### CHECK(제약 조건)
- 특정 컬럼의 값이 지정된 조건을 만족해야 함
- MySQL 8.0 이상에서 지원됨
```MYSQL
CREATE TABLE employees (
	id INT
);
```

##### AUTO_INCREMENT(자동 증가)
-  INT 타입에서 사용 가능하며, **새로운 레코드가 추가될 때 자동 증가**
-  PRIMARY KEY 또는 UNIQUE가 필요
```MYSQL
CREATE TABLE users (
	id INT PRIMARY KEY AUTO_INCREMENT,
	username VARCHAR(50) NOT NULL
);
```
- id 값이 자동으로 1부터 증가
- AUTO_INCREMENT는 하나의 테이블에서 오직 하나의 컬럼에만 사용할 수 있다.

###### 두 개의 자동 증가 값이 필요한 경우
AUTO_INCREMENT를 하나만 유지하면서, **별도의 시퀀스 테이블을 만들어 관리**하는 방법도 있다.
```MYSQL
CREATE TABLE sequence_table (
	seq-id INT AUTO_INCREMENT PRIMARY KEY
);

CREATE TABLE products (
	product_id INT PRIMARY KEY,
	batch_number INT NOT NULL
);
```

```mysql
CREATE TABLE sequence_table ( seq_id INT AUTO_INCREMENT PRIMARY KEY ); CREATE TABLE products ( product_id INT PRIMARY KEY, batch_number INT NOT NULL ); INSERT INTO sequence_table () VALUES (); -- 새로운 시퀀스 값 생성 SET @new_id = LAST_INSERT_ID(); -- 방금 생성된 ID 가져오기 INSERT INTO products (product_id, batch_number) VALUES (@new_id, 101);
```

###### 데이터베이스 모델링
: 개념적 설계(ERD) -> 논리적 설계(RDB 이론 적용) -> 물리적 설계(DB 제품 선택, 자료형, 역정규화)
- 한 명의 고객은 여러 개의 상품을 주문할 수 있다.
- 하나의 상품은 여러 명의 고객으로부터 주문되어진다.

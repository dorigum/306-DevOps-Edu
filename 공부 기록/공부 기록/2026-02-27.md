### Oracle vs MySQL

#### 2. MySQL의 경우
- MySQL은 **Oracle처럼 Inline 제약조건 이름 지정은 지원하지 않는다.**
- 즉, 아래 구문은 **MySQL에서 에러**가 난다.
```mysql
CREATE TABLE test (
    id VARCHAR(20) CONSTRAINT test_id_pk PRIMARY KEY
);
```

- 대신 MySQL은 테이블 수준(Table-Level)에서만 제약 조건 이름을 지정할 수 있다.
```mysql
CREATE TABLE test (
    id VARCHAR(20),
    CONSTRAINT test_id_pk PRIMARY KEY (id)
);
```

- 또는 외래키, 유니크 제약 조건도 마찬가지이다.
```mysql
CREATE TABLE orders (
    order_id INT,
    customer_id INT,
    CONSTRAINT orders_cust_fk FOREIGN KEY (customer_id)
        REFERENCES customers(id)
);
```

---
#### 3. 왜 MySQL은 Inline constraint 이름을 지원하지 않을까?
- MySQL의 **SQL Parser(문법 설계)** 자체가 Oracle, SQL Server와 달리, **컬럼 정의 옆에서 제약 조건 이름을 붙이는 문법을 구현하지 않았기 때문이다.**
- MySQL 개발 철학은 **“간단한 문법”**을 선호하는 편이라, 제약조건 이름이 필요한 경우는 **항상 테이블 수준에서만 명시**하도록 일관성 있게 만든 것이다.
- 그래서 MySQL 매뉴얼에도 **constraint 이름은 table-level 제약조건 구문에서만 가능**하다고 명시되어 있다.

#### 4. 정리
- **Oracle** → inline + table-level 둘 다 제약조건 이름 지정 가능
- **MySQL** → inline에서는 이름 지정 불가 ❌, table-level에서만 가능 ⭕
- 따라서 MySQL에서 제약조건 이름을 꼭 주고 싶다면:
```mysql
CREATE TABLE test (
    id VARCHAR(20),
    CONSTRAINT test_id_pk PRIMARY KEY (id)
);
```
👉 즉, MySQL은 **“제약조건 이름을 붙이려면 무조건 테이블 레벨에서 정의”** 해야 한다.
